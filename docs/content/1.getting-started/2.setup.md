## Install

Install the main package together with the generator. The generator is an optional peer dependency that powers schema inference — install it as a `devDependency` since it is only used in development and during builds.

:::code-group
```bash [pnpm]
pnpm add @outloud/adonis-openapi
pnpm add -D @outloud/adonis-openapi-generator
```

```bash [npm]
npm install @outloud/adonis-openapi
npm install -D @outloud/adonis-openapi-generator
```

```bash [yarn]
yarn add @outloud/adonis-openapi
yarn add -D @outloud/adonis-openapi-generator
```

```bash [bun]
bun add @outloud/adonis-openapi
bun add -D @outloud/adonis-openapi-generator
```
:::

## Configure

Run the AdonisJS configure command. It will register the provider, the ace command, the meta file, and copy a default `config/openapi.ts`.

```bash
node ace configure @outloud/adonis-openapi
```

## Tuyau
`@tuyau/core` package is temporarily required to generate routes declaration.

- Install `@tuyau/core` package.
- Add `generateRegistry()` hook to `adonisrc.ts` according this [tutorial](https://docs.adonisjs.com/guides/frontend/api-client).

This should be removed in the next version of AdonisJS when routes scanning and code generation is available outside dev server.

## Verify

Start the dev server and open:

- `/openapi.json` - generated spec
- `/docs` - Scalar API reference UI

::tip
The spec is regenerated on every request in development, so changes to routes, validators, controllers or models appear immediately on reload.
::

## Build

Generate and cache the spec before deploying:

```bash
node ace openapi:generate
```

See more information about building in [Build](/guide/build) section.
