`@outloud/adonis-openapi` generates an OpenAPI 3.1 specification for your AdonisJS application directly from your code. It hooks into the route registry, inspects controller types via `ts-morph`, and produces a `openapi.json` document plus a hosted [Scalar](https://scalar.com) UI — without requiring you to write OpenAPI by hand.

## Features
- **Zero-config** - Reads your existing routes, validators and controllers
- **Type-driven** - Request and response schemas inferred from TypeScript types
- **Validator support** - VineJS validators converted to request body schemas
- **Lucid models** - Models serialized to schemas using their column definitions and naming strategy
- **Route metadata** - Decorate routes, groups and resources with `.openapi()` for summaries, tags, security, etc.
- **Custom handlers** - Extend the generator to handle your own types
- **Scalar UI** - API reference rendered at `/docs` out of the box
- **Build-time generation** - `node ace openapi:generate` writes a cached spec for production

## How It Works

In development, the spec is generated on every request to `/openapi.json` by walking the AdonisJS routes registry and resolving each route's body, query and response types via the TypeScript compiler.

In production, the cached `.adonisjs/openapi.json` file (created by `openapi:generate`) is served instead — no `ts-morph` overhead at runtime.
