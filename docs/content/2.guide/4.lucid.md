---
title: Lucid Models
description: Lucid models are auto-serialized into OpenAPI components based on their column and relationship definitions.
---

The generator ships with a built-in handler for `LucidRow`. Any controller return type that resolves to a Lucid model (or model array) is converted into a reusable component under `components.schemas`.

## Basic Model

```ts
// app/models/user.ts
import { BaseModel, column } from '@adonisjs/lucid/orm'
import { DateTime } from 'luxon'

export default class User extends BaseModel {
  @column({ isPrimary: true })
  declare id: number

  @column()
  declare name: string

  @column()
  declare email: string

  @column.dateTime({ autoCreate: true })
  declare createdAt: DateTime
}
```

```ts
// app/controllers/users_controller.ts
export default class UsersController {
  async index() {
    return User.all()
  }
}
```

The user model is emitted once under `components.schemas.User` and referenced via `$ref` in the response. `DateTime` columns are serialized as strings.

## Serialized Names

The handler honors the model's `namingStrategy.serializedName` and each column's `serializeAs`:

```ts
@column({ serializeAs: 'first_name' })
declare firstName: string

@column({ serializeAs: null })   // excluded from schema
declare passwordHash: string
```

Set `serializeAs: null` to hide internal columns (passwords, tokens, etc.) from the generated spec — the same flag Lucid already uses to hide them from JSON responses.

## Computed Properties

`@computed()` properties are included in the schema:

```ts
import { computed } from '@adonisjs/lucid/orm'

export default class User extends BaseModel {
  @column() declare firstName: string
  @column() declare lastName: string

  @computed()
  get fullName() {
    return `${this.firstName} ${this.lastName}`
  }
}
```

## Relationships

`@hasOne`, `@hasMany`, `@belongsTo` and `@manyToMany` relationships are detected and emitted using their `serializedName`:

```ts
export default class Post extends BaseModel {
  @column() declare id: number
  @column() declare title: string

  @belongsTo(() => User)
  declare author: BelongsTo<typeof User>

  @hasMany(() => Comment)
  declare comments: HasMany<typeof Comment>
}
```

- `author` → single object referencing `User`
- `comments` → array referencing `Comment`

Nullable relations are wrapped in `oneOf: [Model, { type: 'null' }]`.

::note
Relationships are only included when the controller actually loads them (e.g. via `.preload('author')`) and the controller's return type reflects them.
::

## Required vs Optional

A column is marked `required` unless its TypeScript declaration is optional (`?` or `| undefined`). Override Lucid's `declare` to make this explicit:

```ts
@column()
declare email: string             // required

@column()
declare middleName?: string       // optional
```

## Customizing the Schema

To take full control of how a model is serialized, add a `toOpenAPI` method. The generator uses its return type instead of the column definitions:

```ts
export default class User extends BaseModel {
  // ...

  toOpenAPI(): { id: number, displayName: string } {
    return { id: this.id, displayName: this.name }
  }
}
```

This is helpful when the JSON shape diverges from the database shape (e.g. when you compose multiple fields in `serialize()`).
