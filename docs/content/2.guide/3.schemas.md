---
title: Schema Inference
description: How request bodies, query parameters and responses are inferred from your code.
---

The generator walks the AdonisJS route registry and uses `ts-morph` to read each route's body, query and response TypeScript types. You generally do not need to write schemas by hand — write idiomatic AdonisJS code and they appear.

## Path Parameters

Path parameters are derived directly from the route pattern.

```ts
router.get('/users/:id/posts/:postId?', [PostsController, 'show'])
```

Produces:

```yaml
parameters:
  - name: id
    in: path
    required: true
    schema: { type: [string, integer] }
  - name: postId
    in: path
    required: false
    schema: { type: [string, integer] }
```

`?` in the pattern marks the parameter optional.

## Request Body

When your controller validates the request with a VineJS validator, the generator picks up the validator type and converts it into a JSON Schema for the request body.

```ts
// app/validators/user.ts
import vine from '@vinejs/vine'

export const createUserValidator = vine.crate({
  name: vine.string().minLength(1),
  email: vine.string().email(),
  age: vine.number().optional(),
})
```

```ts
// app/controllers/users_controller.ts
export default class UsersController {
  async store({ request }: HttpContext) {
    const data = await request.validateUsing(createUserValidator)
    return User.create(data)
  }
}
```

Produces a `requestBody` with `application/json` content matching the validator schema. The validator is referenced by name (e.g. `CreateUserValidator`) and emitted under `components.schemas.CreateUserProps`, so it is reused across operations.

::tip
The generator strips a trailing `Validator` from the export name and suffixes `Props` for the component name (e.g. `createUserValidator` → `CreateUserProps`).
::

### Multipart / File Uploads

If any property in the validator is a `MultipartFile`, the request body content type is automatically switched to `multipart/form-data`:

```ts
export const uploadValidator = vine.compile(
  vine.object({
    title: vine.string(),
    file: vine.file({ size: '10mb', extnames: ['png', 'jpg'] }),
  }),
)
```

```yaml
requestBody:
  content:
    multipart/form-data:
      schema:
        type: object
        properties:
          title: { type: string }
          file: { type: string, format: binary }
```

## Query Parameters

Query parameters come from the type of `request.qs()` or a query validator. Each top-level property is emitted as an individual `in: query` parameter:

```ts
export const listUsersValidator = vine.compile(
  vine.object({
    limit: vine.number().optional(),
    search: vine.string().optional(),
  }),
)

export default class UsersController {
  async index({ request }: HttpContext) {
    const { limit, search } = await request.validateUsing(listUsersValidator)
    // ...
  }
}
```

Produces:

```yaml
parameters:
  - name: limit
    in: query
    required: false
    schema: { type: number }
  - name: search
    in: query
    required: false
    schema: { type: string }
```

When a query validator is a union of multiple shapes, properties with the same name are merged into a `oneOf` schema.

## Response

The response schema is inferred from the controller method's return type. If you return a Lucid model (or model array), the model is serialized — see [Lucid Models](/guide/lucid). For plain objects, the TypeScript type is converted directly:

```ts
export default class UsersController {
  async show({ params }: HttpContext): Promise<{ id: number, name: string }> {
    return { id: params.id, name: 'John' }
  }
}
```

Produces:

```yaml
responses:
  200:
    description: OK
    content:
      application/json:
        schema:
          type: object
          properties:
            id: { type: number }
            name: { type: string }
          required: [id, name]
```

::note
Only the `200` response is inferred. Document other status codes via `.openapi({ responses: { ... } })` — see [Route Metadata](/guide/routes#documenting-error-responses).
::

## `toOpenAPI` / `toJSON` Overrides

Any class with a `toOpenAPI` (preferred) or `toJSON` method has its schema taken from that method's return type. This is the easiest way to control how a class is serialized without writing a handler:

```ts
class Money {
  constructor(public amount: number, public currency: string) {}

  toOpenAPI(): { amount: number, currency: string } {
    return { amount: this.amount, currency: this.currency }
  }
}
```

`Money` is serialized as `{ amount: number, currency: string }` wherever it appears.
