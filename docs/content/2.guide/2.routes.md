---
title: Route Metadata
description: Attach OpenAPI metadata to routes, route groups and resources with the .openapi() macro.
---

The provider extends `Route`, `RouteGroup` and `RouteResource` with an `.openapi()` macro. Use it to attach any [OpenAPI Operation Object](https://spec.openapis.org/oas/v3.1.0#operation-object) — summary, description, tags, security, responses, etc.

::tip
You only need `.openapi()` for metadata that cannot be inferred from types. Request bodies, query parameters, path parameters and responses are detected automatically. See [Schema Inference](/guide/schemas).
::

## Route

```ts
import router from '@adonisjs/core/services/router'

router.get('/users/:id', [UsersController, 'show'])
  .openapi({
    summary: 'Get user by ID',
    description: 'Returns the user matching the given ID.',
    tags: ['Users'],
  })
```

The metadata is merged into the generated operation — your fields win over inferred ones, so you can override the auto-detected request body or responses if needed:

```ts
router.post('/files', [FilesController, 'upload'])
  .openapi({
    summary: 'Upload file',
    requestBody: {
      required: true,
      content: {
        'multipart/form-data': {
          schema: {
            type: 'object',
            properties: { file: { type: 'string', format: 'binary' } },
          },
        },
      },
    },
  })
```

## Route Group

`.openapi()` on a group is merged into every nested route. Per-route metadata wins over group metadata.

```ts
router.group(() => {
  router.get('/me', [MeController, 'show'])
  router.patch('/me', [MeController, 'update'])
})
  .prefix('/api/v1')
  .openapi({
    tags: ['Account'],
    security: [{ bearer: [] }],
  })
```

Groups support nested resources and groups — metadata propagates through all levels.

## Route Resource

`.openapi()` on a resource accepts both top-level fields (applied to every action) and per-action overrides keyed by action name:

```ts
router.resource('posts', PostsController)
  .openapi({
    tags: ['Posts'],
    security: [{ bearer: [] }],

    index: { summary: 'List posts' },
    show: { summary: 'Get post by ID' },
    store: { summary: 'Create post' },
    update: { summary: 'Update post' },
    destroy: { summary: 'Delete post' },
  })
```

Action keys: `index`, `show`, `store`, `update`, `destroy`, `create`, `edit`.

## Common Patterns

### Security

```ts
router.get('/admin/stats', [AdminController, 'stats'])
  .openapi({
    summary: 'Admin stats',
    tags: ['Admin'],
    security: [{ bearer: [] }],
  })
```

Pair this with a `securitySchemes` entry in your `document.components`:

```ts
defineConfig({
  document: {
    components: {
      securitySchemes: {
        bearer: { type: 'http', scheme: 'bearer' },
      },
    },
  },
})
```

### Deprecated Endpoints

```ts
router.get('/legacy/users', [LegacyController, 'list'])
  .openapi({
    deprecated: true,
    summary: 'List users (deprecated, use /users)',
  })
```

## Excluding Routes

Combine `.openapi({ summary })` with `generator.routes: 'auto'` to opt-in only the documented routes:

```ts
// config/openapi.ts
generator: { resolve: import.meta.resolve, routes: 'auto' }

// start/routes.ts
router.get('/users', [UsersController, 'index'])
  .openapi({ summary: 'List users' })   // included

router.get('/internal/ping', () => 'pong') // excluded
```

Or list patterns explicitly via `routes.include` / `routes.exclude` — see [Configuration](/guide/configuration#options).
